Theory

Agents are specialized services that handle specific tasks like checking data, applying business rules, or creating content. They work by talking to LLMs or external APIs. Each agent uses clear input and output formats (usually JSON), which makes them modular, easier to test, and simple to connect with other systems. By linking several agents together, we can build applications with multiple steps.

In this project, we're creating an AI Diet & Meal Planner - a FastAPI app with multiple Agents that makes meal planning much easier. Our system will use three agents:

  • Inventory agent, which processes and checks the food items you already have

  • Diet agent, which filters ingredients based on your dietary needs (like vegan or keto)

  • Planner agent, which creates complete, ready-to-cook recipes using your available ingredients

The whole system will work as connected FastAPI endpoints that talk to each other through JSON. The goal is to take away the daily headache of figuring out what to cook by giving you personalized recipes based on what you like and what ingredients you have on hands.

Description

Before we get to the agents, we need to set up a working FastAPI application with a clear structure and a simple health-check endpoint. This be used as a base for our application.

In this stage, your job is to create a simple FastAPI service with a health-check endpoint (GET/) that returns a JSON response showing the server is working properly. This step makes sure your development setup is configured correctly, setting you up for the more complex features we'll add later.

Make sure the server is running and has the described logic implemented before submitting each stage for checking.

Objectives

  • Set up a FastAPI service with a AI Diet & Meal Planner title.

  • Create a single root endpoint (GET/) that returns the following JSON: {"message": "Success!"}.

  • Make sure the FastAPI server can run with Uvicorn on port 8000 using auto-reload mode.

  • Check that the service correctly handles and returns JSON responses.

Example

Request:

# sample curl request
curl -i http://localhost:8000/

Expected Response:

HTTP/1.1 200 OK
content-type: application/json
{"message":"Success!"}

To submit this stage for a test, you have to run the FastAPI server from the terminal with the following command (here, main is the name of the file, main.py, and app is the previously defined name of the FastAPI server):

uvicorn main:app --reload

Once the server is running, you can submit the solution. Note that the test assumes the FastAPI server is running on localhost and port 8000.
